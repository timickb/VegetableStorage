# Склад овощей (какая-то дичь с пиргрейда на ПИ)

Программа делает следующую вещь: запрашивает у пользователя информацию о некотором складе,
запрашивает некоторые действия над этим складом, после чего выводит результат - состояние
склада после применения к нему этих действий.

Описанием склада является:
* Максимальное число контейнеров, которое он может вместить (целое положительное число)
* Цена хранения одного контейнера в тугриках (целое положительное число)

Контейнером является сущность, содержащая поля:
* Список ящиков, помещенных в этот контейнер
* Максимальный суммарный вес ящиков в килограммах
* Идентификатор - произвольная строка длиной до 64 символов.

Действием является сущность, содержащая поля:
* Наименование действия. Их всего два: `add` (добавить контейнер) и `remove` (удалить контейнер).
* Аргумент действия - идентификатор контейнера. Обязателен для действия `remove`, опционален для действия `add` - по умолчанию
контейнеру будет присовен идентификатор, равный порядковому номеру этого контейнера относительно ВСЕХ добавленных контейнеров
(т.е. даже тех, которым было отказано в помещении на склад).

Действие "add" не приводит к результату в том случае, если хранение предлагаемого контейнера нерентабельно.

Параметры склада, контейнеры и действия могут быть запрошены как в интерактивном режиме через консоль, так и путем загрузки
трех JSON файлов:
* Описание склада. Шаблон находится в `examples/storage_description.json`
* Список контейнеров. Шаблон находится в `examples/containers.json`
* Список действий. Шаблон находится в `examples/actions.json`

В конце программа предложит сохрнаить результат применения действий к складу в JSON файл.
